\# AGENTS.md (repo-wide, long-lived)



\## 项目目标（长期）

构建“集成大模型的个性化学习辅助系统”：文档资料管理、摘要/知识点提炼、基于资料问答（RAG）、自动测验、学习进度与画像。



\## 当前阶段说明（可变）

\- 每次任务开始时，\*\*先阅读根目录 CURRENT.md\*\*（或 PLANS.md），以它作为本次工作的验收标准与优先级。



\## 技术栈偏好（可调整，但要解释）

\- 后端：Python + FastAPI

\- 前端：Vue 3 + Vite

\- 检索：先轻量可用（关键词/TF-IDF），后续再升级向量检索（Chroma/FAISS）

\- LLM：支持可选 API Key；无 Key 时必须有降级（mock）路径，确保演示不崩



\## 交付与质量门槛（长期有效）

\- 每次改动必须：

&nbsp; 1) 能启动（后端/前端）

&nbsp; 2) README 更新（启动命令 + 演示路径 + 关键配置）

&nbsp; 3) 失败要有用户可理解的错误提示（前端不白屏、后端不抛裸异常）

\- 依赖最小化：新增依赖前先说明“为什么必须 + 替代方案”，再执行

\- 不提交密钥：使用 .env.example；日志里不要输出敏感信息



\## 工作方式（Codex执行规则）

\- 开始先给一个不超过 12 条的 TODO 计划

\- 过程中尽量小步提交；关键节点提供可运行命令

\- 若环境缺少依赖/网络受限导致安装失败，优先选择“更少依赖的实现方案”，并在 PR/说明中写清限制与下一步



\## 目录结构建议（可按需创建）

\- backend/    后端服务

\- frontend/   前端应用

\- data/       本地上传/解析产物（应加入 .gitignore）

\- docs/       文档与阶段计划（可放 CURRENT.md/PLANS.md）
