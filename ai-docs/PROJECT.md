# 项目定义（AI 必读）

## 1. 这是什么项目

本项目是一个 **集成大型语言模型（LLM）的个性化学习辅助系统**，目标是构建一个**可实际使用**的学习工具，而不是仅用于展示的 Demo。

系统核心能力（以“可用”为标准）：
- 学习资料管理：上传、解析、存储（PDF / Word / Markdown）
- 内容理解与检索：分段（chunk）+ 向量化 + FAISS 检索
- 基于资料的智能问答：RAG（检索增强生成），回答应优先基于资料
- 自适应测验：基于学习者画像控制出题难度与覆盖范围
- 学习记录：记录问答/测验行为，为画像与难度控制提供数据

## 2. 这不是什么项目（很重要）

本项目不是：
- 通用聊天机器人（不追求“什么都能聊/都能答”）
- 纯内容生成工具（不以“生成越多越好”为目标）
- 需要训练模型或复杂机器学习的研究项目（默认不做训练/微调）
- 追求炫技架构的工程项目（不做不必要的分布式/微服务）

## 3. 技术栈与基本约束（稳定约束）

未经明确指示不得更改：
- 后端：FastAPI（Python）
- 数据库：MySQL
- ORM / 迁移：SQLAlchemy + Alembic
- 向量检索：FAISS
- 开发方式：纵向切片（先打通一条链路再扩展）

## 4. 成功标准（稳定标准）

当满足以下条件时，认为项目方向正确且“可用”：
- 系统行为可预测、可解释：尤其是测验难度不会无故“乱跳”
- 低水平学习者不会频繁被高难题打击信心（默认策略偏保守：易→难）
- 上传→解析→索引→检索→问答/测验 的核心流程可连续稳定运行
- 重启服务后功能仍可用，数据与索引可恢复或可重建
